RIG Notes

Notes for 0.6.3.5:

  • This version introduces some new (minor but useful) features. It seems pretty solid and is a good candidate to become 0.6.4-stable
  • Albums has Grid vs Vertical layout, displays date of modification, etc.
  • New "ignore lists" for albums/images (patterns of files to be ignored)

Notes for 0.6.3.4:

  • This is an intermediary "dev" release (only CVS, no specific tag nor file download)
  • Introduced support for videos
  • Fixed broken album frame for Mozilla 1.0
  • Support in-album "file_info.diz" or "descript.ion" files

Notes for 0.6.3.3:

  • Japanese translation completed by INOUE Tatsuo
  • Some cleanup and fixes

Notes for 0.6.3.2:

  • Misc fixes
  • Preparation for upload feature
  • Ability to translate UI via web interface (experimental)

Notes for 0.6.3.1:

  • Mostly untested, but few problems are to be expected (modifications since 0.6.3.0 are fairly limited).

Notes for 0.6.3.0:

  • Seems fairly stable, although there's one known bug in the URL-rewrite option (fails for cookies).
  • The options are now stored in the rig-options subdir instead of the rig-cache subdir. Consequently, you can either rebuild all options for your site or use the hidden "fix options" function (it will read the old one and write new ones, but it may fail to get the correct default thumbnails stored path, since the format changed).

RIG Change Log

New in 0.6.3.5:

  • Translation of dates in web pages
  • Albums: Display album modification date in grid layout
  • Albums: Grid vs Vertical layout
  • Feature: "ignore lists" for albums/images (patterns of files to be ignored)

New in 0.6.3.4:

  • Introduced support for videos (display and rig_thumbnail.exe)
  • Added movie frame to video thumbnails
  • Experimental: Class layout
  • Fixed broken album frame for Mozilla 1.0 (render using cell background images)
  • Started adding some useful(?) comments to settings/prefs.php
  • Support in-album "file_info.diz" or "descript.ion" files
  • New makefiles for rig_thumbnail.exe

New in 0.6.3.3:

  • Removed unused language strings. Added new rig_display_back_to_album method
  • Disabled admin_image (not finished -- experimental)
  • Lang: Update jp/fr/es strings to match en, removed unused strings. Using latest jp translation file.
  • Fixed album/image count display in admin mode
  • Added "old" layout for image display, with image layout pref variable.

New in 0.6.3.2:

  • Album borders: ability to deactivate for Mozilla 1.0 users
  • Translate: deactivated by default
  • Translate: ability to update the language files from the web interface
  • Upload directories (not used yet)
  • Pref: can customize <meta name="robots" ...>
  • Admin: admin-image vs admin-album pages

New in 0.6.3.1:

  • Login: plain vs crypt vs MD5 password in the password file
  • Login: display error msg for invalid user/password
  • Themes: Added gray theme color

New in 0.6.3:

  • Display copyright in image page, display number of images/albums in tables
  • Hidden fix_option in admin page to convert option.txt from 0.6.2 to 0.6.3 (experimental)
  • Using rig_options directory
  • Renamed src function with rig_ prefix everywhere
  • Only display phpinfo if _debug_ enabled or admin mode

New in 0.6.2.4:

  • Stable enough to become 0.6.3 (need file package on SourceForge)
  • URL-Rewrite support (cf site/prefs.php -- has at least one known bug)

New in 0.6.2.3:

  • Tuned album thumbnail table
  • Charset and html lang support

New in 0.6.2.2:

  • Better error reporting
  • Multiple languages support (SP+JP on top of EN+FR)
  • Theme pages support
  • jhead support
  • Grayscale JPEG support
