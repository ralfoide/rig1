RIG Notes

Notes in 1.0:

  • This is the last final version. It fixes a security vulnerability in the entry point that allows for PHP code inclusion and cross-site vulnerability. All previous version are unsafe to use.

Notes in 0.7.4:

  • This is an intermediary dev release, with Polish translation by Alfred Broda and some minor fixes.

Notes in 0.7.3:

  • This is an intermediary dev release, with one minor fix and one minor feature.

Notes in 0.7.2:

  • This is an intermediary dev release, with some fixes & features.
  • This version adds caching to the image & overview pages and eventually fixes a dumb bug that was always invalidating cached pages when the cache was reloaded.
  • A new most significant feature is streaming of resized images via PHP instead of letting Apache serve the images directly from the cache folders.
  • Consequently, it is also now possible to avoid placing all caches and sources images under the document root. Instead data can be located out of reach of Apache.
  • Important: When you upgrade to this version, you must review your location.php file to adjust the data paths!

Notes for 0.7.1:

  • This is an intermediary dev release, with one minor fix.

Notes for 0.7:

  • This is a stable version.
  • License has been changed to a pure GNU General Public License v2.
  • There are numerous little, insignificant and time-consuming fixes compared with the latest stable version 0.6.4.4. Upgrading is recommended.

Notes for 0.6.4.5:

  • This is an intermediary dev release, with some minor but useful fixes.

Notes for 0.6.4.4:

  • This is an intermediary dev release, with some minor but useful features.
  • Although considered non-stable (low testing and new features to be added soon), this release seems to perform very well.

Notes for 0.6.4.3:

  • This is an intermediary dev release, with some minor but useful features.

Notes for 0.6.4.2:

  • This is an intermediary dev release, with some minor but useful features.
  • Some preferences have been added to enable or disable features (album descriptions, album cache, viewing of hidden items.)
  • The location.php and htdocs have been updated to be more flexible.
  • A new folder is used to move admin sources out of the normal sources.
  • Although considered non-stable (low testing and new features to be added soon), this release seems to perform very well.

Notes for 0.6.4.1:

  • This is a stable version.
  • This version does not technically add any feature, but it fixes a bunch of small and annoying details.
  • The code has been updated to comply with PHP 4.3.2 installation defaults: it works with register_globals=off and allow_call_time_pass_reference=off.
  • This version also has been tested with IIS 5 and PHP 4.3.2 on a Windows box.

Notes for 0.6.4.0:

  • This version features a completly rewritten installation guide. Altough I consider the installation guide to be always obsolete (since it is always updated after features are posted online), its reading should help a lot people who are new to RIG. It covers how to compile the thumbnail application and how to configure a basic site, including tricky parts like directories access rights.
  • This version features a brand new album HTML cache generation: each time an album page is rendered, RIG keeps a cache of it and serves the cached version next time the same unchanged album is viewed. This greatly reduces the load of the server.
  • This version was supposed to be the next stable one but further testing showed this was premature

Notes for 0.6.3.5:

  • This version introduces some new (minor but useful) features. It seems pretty solid and is a good candidate to become 0.6.4-stable
  • Albums has Grid vs Vertical layout, displays date of modification, etc.
  • New "ignore lists" for albums/images (patterns of files to be ignored)

Notes for 0.6.3.4:

  • This is an intermediary "dev" release (only CVS, no specific tag nor file download)
  • Introduced support for videos
  • Fixed broken album frame for Mozilla 1.0
  • Support in-album "file_info.diz" or "descript.ion" files

Notes for 0.6.3.3:

  • Japanese translation completed by INOUE Tatsuo
  • Some cleanup and fixes

Notes for 0.6.3.2:

  • Misc fixes
  • Preparation for upload feature
  • Ability to translate UI via web interface (experimental)

Notes for 0.6.3.1:

  • Mostly untested, but few problems are to be expected (modifications since 0.6.3.0 are fairly limited).

Notes for 0.6.3.0:

  • Seems fairly stable, although there's one known bug in the URL-rewrite option (fails for cookies).
  • The options are now stored in the rig-options subdir instead of the rig-cache subdir. Consequently, you can either rebuild all options for your site or use the hidden "fix options" function (it will read the old one and write new ones, but it may fail to get the correct default thumbnails stored path, since the format changed).
